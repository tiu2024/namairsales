# Impeccable Design Context — NamAirSales

## Design Context

### Users
Internal office workers (5–10 people) — accountants and salesmen — at desktop PCs throughout the workday. They use this app repeatedly for core job tasks: logging sales, tracking balances, reviewing payments. The interface should feel reliable and easy to scan quickly, not require cognitive overhead. Errors and financial states must be instantly legible.

### Brand Personality
Friendly, clear, approachable — not corporate or intimidating. The app handles money and business data, so it must feel trustworthy and accurate, but it should also feel easy to use without training. Think: a tool a colleague built for you, polished but not flashy.

### Aesthetic Direction
- Clean light mode with a warm, approachable tone (not cold/sterile)
- Dark mode supported — navy sidebar already suggests a dark-mode-friendly palette
- No specific reference app; current direction (navy + emerald + crimson on white) is the foundation
- Typography-forward: Inter for readability, JetBrains Mono for data scannability
- Semantic color use: navy = actions/brand, emerald = positive/linked, crimson = debt/error
- Avoid: heavy drop shadows, decorative gradients, excessive animation, anything that feels like a generic SaaS landing page

### Design Tokens

**Colors**
```css
--navy:        #1B3A6B   /* primary, sidebar, buttons */
--navy-light:  #254d8f   /* button hover */
--emerald:     #059669   /* positive balance, linked */
--crimson:     #DC2626   /* debt, error, pending */
--bg:          #F8FAFC   /* page background */
--surface:     #FFFFFF   /* cards, modals */
--border:      #E2E8F0   /* dividers, inputs */
--text:        #1E293B   /* primary text */
--text-muted:  #64748B   /* labels, hints */
```

**Typography**
- Body: Inter 400/500/600, 14px base
- Data (currency, quantities, dates): JetBrains Mono 400/500, 13px
- Headings: Inter 600, 16–20px
- Table headers: Inter 600, 12px, uppercase, 0.04em letter-spacing

**Spacing** — 4px base unit; key values: 4, 8, 10, 12, 16, 20, 24, 32, 40px

**Border radius** — inputs/buttons: 6px; cards: 8px; login card: 12px; badges: 99px

**Layout** — Fixed sidebar 240px; collapses to 60px icon rail at 900px; main content 32px padding

### Design Principles

1. **Clarity over cleverness** — Every UI decision should make the data easier to read and act on. No decoration that doesn't carry meaning.
2. **Semantic color is sacred** — Navy, emerald, and crimson have defined meaning (brand/action, positive, negative). Never repurpose them.
3. **Data deserves monospace** — All financial amounts, quantities, and dates in tables use JetBrains Mono for alignment and scannability.
4. **Friendly, not flashy** — Approachable tone: comfortable spacing, legible type, warm neutrals. Avoid corporate coldness or over-polished SaaS aesthetics.
5. **Desktop-first, but not desktop-only** — Primary context is desktop PC. Tablet breakpoint at 900px is a minimum bar; don't break layout at common desktop widths.
6. **Dark mode as a first-class concern** — When adding or refining components, consider both light and dark mode. The navy sidebar is already dark — extend this thinking.
